# Awesome Textualize Projects

A curated collection of community-built applications using Textual, Rich, and Rich CLI frameworks.

## Overview

This is a comprehensive list of third-party applications demonstrating real-world Textual usage patterns. These projects showcase the versatility of the framework for building everything from simple utilities to full-featured applications.

**Repository Stats** (as of 2024-03-18):
- **93 stars** on GitHub
- Community-curated collection
- Covers diverse application domains

## Project Categories

### Development Tools

**Code Inspection & Debugging**:
- **[Shira](https://github.com/darrenburns/shira)** - The python inspector üîç. Inspect Python objects interactively.
- **[Astview](https://github.com/davep/textual-astview)** - Textual-based Python AST viewing widget library and application. Visualize Python abstract syntax trees.
- **[Dunk](https://github.com/darrenburns/dunk)** - Prettier git diffs in the terminal üé®. Enhanced diff visualization.

**Database & Query Tools**:
- **[Harlequin](https://pypi.org/project/harlequin/)** - The SQL IDE for Your Terminal. Full-featured SQL development environment.
- **[Rexi](https://github.com/royreznik/rexi)** - Terminal UI for Regex Testing. Interactive regex pattern testing and validation.

**Web & API Development**:
- **[Django-TUI](https://github.com/anze3db/django-tui)** - Inspect and run Django Commands in a text-based user interface.
- **[Elia](https://github.com/darrenburns/elia)** - A terminal ChatGPT client built with Textual. AI assistant in your terminal.

### System Administration & DevOps

**Log Management**:
- **[Toolong](https://github.com/Textualize/toolong)** - Official Textualize app to view, tail, merge, and search log files (plus JSONL). Advanced log analysis.
- **[Logmerger](https://github.com/ptmcg/logmerger)** - TUI utility to view multiple log files with merged timeline. Correlate events across logs.

**System Utilities**:
- **[RecoverPy](https://github.com/PabloLec/RecoverPy)** - Interactively find and recover deleted or overwritten files from your terminal. Data recovery tool.
- **[SSHaMan](https://github.com/cornyhorse/sshaman)** - SSH Management. Manage SSH connections and configurations.
- **[HumBLE Explorer](https://github.com/koenvervloesem/humble-explorer)** - Cross-platform, command-line and human-friendly Bluetooth Low Energy Scanner.

### File & Content Management

**File Browsers & Pickers**:
- **[Kupo](https://github.com/darrenburns/kupo)** - A terminal file browser, kupo! Navigate filesystem with keyboard shortcuts.
- **[Fspicker](https://github.com/davep/textual-fspicker)** - Textual widget library for picking things in the filesystem. Reusable file selection component.

**Document Viewers**:
- **[Markdown Browser](https://github.com/willmcgugan/textual-markdown)** - Markdown in the terminal. View formatted markdown documents.
- **[Frogmouth](https://github.com/Textualize/frogmouth)** - Official Textualize Markdown browser for your terminal. Enhanced markdown viewing experience.
- **[Imageview](https://github.com/adamviola/textual-imageview)** - Simple terminal-based image viewer. Display images in the terminal.

**Data Visualization**:
- **[Jtree](https://github.com/oleksis/jtree)** - CLI for displaying JSON in a tree view. Hierarchical JSON exploration.
- **[textual-plotext](https://pypi.org/project/textual-plotext/)** - Textual widget wrapper for the Plotext plotting library. Terminal-based charts and graphs.

### Productivity & Organization

**Note-Taking & Bookmarks**:
- **[NoteSH](https://github.com/cvaniak/notesh)** - Fully functional sticky notes App in your Terminal. Quick note-taking interface.
- **[Tinboard](https://pypi.org/project/tinboard/)** - A Pinboard client for the terminal. Manage bookmarks from CLI.

### Creative & Entertainment

**Games**:
- **[FivePyFive](https://github.com/davep/fivepyfive)** - An annoying puzzle for the terminal; built with Textual. Puzzle game implementation.
- **[Conways Game of Life](https://github.com/thomascrha/textual-game-of-life)** - Implementation of Conway's game of life (cellular automata) using Textual.
- **[Usolitaire](https://github.com/eliasdorneles/usolitaire)** - Solitaire in your terminal, powered by Unicode. Classic card game.

**Creative Tools**:
- **[Pain](https://github.com/1j01/textual-paint)** - üé® MS Paint.. in your terminal. Pixel art and drawing tool.
- **[Upiano](https://github.com/eliasdorneles/upiano)** - A Piano in your terminal. Musical instrument simulation.
- **[Palettepal](https://github.com/cdelker/palettepal)** - Terminal-Based Color Editor and Palette Generator. Design color schemes.
- **[Rich Pixels](https://github.com/darrenburns/rich-pixels)** - Rich-compatible library for writing pixel images and ASCII art to the terminal.

### Reusable Widgets & Components

**Input Components**:
- **[Autocomplete](https://github.com/darrenburns/textual-autocomplete)** - Easily add autocomplete dropdowns to your Textual apps. Dropdown widget with search.
- **[Select](https://github.com/mitosch/textual-select)** - Simple drop-down (select) for Textual with optional search functionality.
- **[Datepicker](https://github.com/mitosch/textual-datepicker)** - A datepicker for textual. Calendar date selection widget.

**Display Components**:
- **[Canvas](https://github.com/davep/textual-canvas)** - Simple character-based canvas widget for use with Textual. Drawing surface.
- **[Terminal](https://github.com/mitosch/textual-terminal)** - Terminal emulator widget for Textual. Embed terminal in your app.

**Utility Components**:
- **[Textual Query Sandbox](https://pypi.org/project/textual-query-sandbox/)** - Simple tool for displaying terminal dimensions. Development utility.

## Common Use Cases & Patterns

### 1. **Developer Productivity Tools**
Pattern: Code inspection, debugging, and visualization tools that enhance development workflow.
Examples: Shira (inspector), Astview (AST viewer), Dunk (git diffs)

### 2. **Data Exploration & Visualization**
Pattern: Interactive viewers for structured data formats (JSON, SQL, logs).
Examples: Jtree (JSON), Harlequin (SQL), Toolong (logs)

### 3. **File System Navigation**
Pattern: Keyboard-driven file browsers and pickers for efficient navigation.
Examples: Kupo (browser), Fspicker (picker widget)

### 4. **System Administration**
Pattern: SSH management, Bluetooth scanning, file recovery - sysadmin utilities.
Examples: SSHaMan, HumBLE Explorer, RecoverPy

### 5. **Reusable Widget Libraries**
Pattern: Modular components that can be integrated into other Textual apps.
Examples: Autocomplete, Datepicker, Canvas, Terminal emulator

### 6. **Creative & Entertainment**
Pattern: Games and creative tools demonstrating Textual's versatility beyond utilities.
Examples: Pain (MS Paint clone), Usolitaire, Upiano

### 7. **AI & Modern APIs**
Pattern: Integration with external services (ChatGPT, databases, web APIs).
Examples: Elia (ChatGPT), Harlequin (SQL databases)

## Key Insights

### Application Domains
- **Developer Tools**: Largest category, demonstrating Textual's strength for dev productivity
- **System Utilities**: Strong showing for sysadmin tasks (logs, SSH, file recovery)
- **Creative Tools**: Surprising breadth (games, music, art) showing framework flexibility
- **Reusable Components**: Healthy ecosystem of shareable widgets

### Architecture Patterns
- **Standalone Applications**: Complete apps with their own CLI (Elia, Harlequin, Toolong)
- **Widget Libraries**: Reusable components for integration (Autocomplete, Canvas, Datepicker)
- **Hybrid Approach**: Libraries that include demo apps (Fspicker, Astview)

### Common Features
- **Keyboard Navigation**: Nearly all apps emphasize keyboard-driven interfaces
- **Search Functionality**: Frequent pattern for filtering/finding content
- **Real-time Updates**: Many apps handle live data (logs, terminals, file systems)
- **Configuration**: Support for themes, keybindings, and user preferences

### Technology Integration
- **External APIs**: ChatGPT (Elia), Pinboard (Tinboard)
- **System Services**: Bluetooth (HumBLE), SSH (SSHaMan)
- **File Formats**: JSON, SQL, Markdown, images, logs
- **Other Libraries**: Plotext (plotting), AST parsing, regex engines

## Learning Resources

For developers building Textual apps:

1. **Widget Development**: Study `textual-autocomplete`, `textual-select`, `textual-datepicker` for reusable component patterns
2. **File System Apps**: Examine `kupo` and `fspicker` for navigation and selection UX
3. **Data Visualization**: Check `jtree` and `textual-plotext` for displaying structured data
4. **Real-time Apps**: Analyze `toolong` and `textual-terminal` for handling live data streams
5. **Complex Applications**: Study `harlequin` and `elia` for production-ready app architecture
