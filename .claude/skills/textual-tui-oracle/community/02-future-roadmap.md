# The Future of Textual - Roadmap and Direction

## Overview

Textual transitions from a commercial venture to a community-driven open source project while maintaining its position as a mature, battle-tested TUI framework.

**Key Insight**: Textual originated as a hobby project built on Rich, evolved into a full-fledged commercial endeavor, and is now returning to its open source roots with continued maintenance and development.

---

## Company Transition (May 2025)

**Textualize, Inc. Status**: Company wrapping up operations in coming weeks

**Why the Transition**:
- Textual was "a solution in search of a problem"
- Despite many use cases where Textual is a fantastic solution, no shared pain-point was found to build a viable business around
- The framework itself succeeded; the business model did not

## Project Maturity Assessment

**Current State**: Mature and battle-tested
- Software is never finished, but Textual has reached production-ready stability
- Proven in real-world applications across diverse use cases
- Robust community of developers building TUIs and web apps (via textual-serve)

**Development Philosophy**:
> "Working on Textual has been a constant source of delight; figuring out how to make a terminal do things that it shouldn't really be able to do"

**Technical Achievements**:
- Overcame baffling edge cases in emulators and terminal protocols
- Built CSS-based styling system for terminal apps
- Created comprehensive widget library
- Enabled web app deployment via textual-serve

## Historical Context

**Origin Story** (November 2021):
Textual started as a hobby project exploring terminal capabilities beyond what should be possible. Early tweet from Will McGugan:

> "Making good progress with Textual CSS. Here's a 'basic' app. The Python + CSS in the screenshots generates the layout in the terminal here. Separating the layout and design from the runtime logic will make it easy to create gorgeous TUI apps. ðŸ¤©"

**Evolution Timeline**:
1. **Hobby Project** (2021) - Built on Rich, exploring terminal capabilities
2. **Framework Development** - CSS system, widgets, layout engine
3. **Commercial Venture** (Textualize, Inc.) - Attempted to build business around framework
4. **Community Project** (2025+) - Return to open source roots with continued maintenance

## Philosophy and Vision

**Core Belief**: Making terminals do things they shouldn't be able to do
- Push boundaries of terminal capabilities
- Overcome protocol limitations creatively
- Maintain elegance and usability

**Technical Approach**:
- Separate layout/design from runtime logic
- CSS-based styling for terminal apps
- Comprehensive widget library
- Cross-platform terminal support

**Community Values**:
- Open source first
- Excellent documentation
- Developer delight
- Real-world problem solving

Textual's journey from hobby project to commercial venture to community project demonstrates both the framework's technical success and the challenges of building businesses around developer tools. The maturity and quality of the framework ensure its continued relevance in the TUI ecosystem.
