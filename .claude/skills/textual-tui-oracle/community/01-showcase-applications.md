# Textual Showcase Applications

Real-world applications built with Textual, demonstrating the framework's versatility and power across different domains.

## Overview

The Textualize projects showcase (https://www.textualize.io/projects/) features 12 diverse applications spanning database management, developer tools, system monitoring, content viewers, and even creative applications like a terminal piano. These showcase the breadth of use cases Textual enables.

---

## Featured Applications

### Database & Development Tools

#### Dolphie
**Purpose**: Real-time MySQL monitoring
**Description**: An intuitive feature-rich terminal tool for monitoring MySQL databases in real-time
**Repository**: https://github.com/charles-001/dolphie
**Use Case**: Database administration, performance monitoring
**Key Pattern**: Real-time data visualization and metrics display

#### Harlequin.sh
**Purpose**: DuckDB IDE for terminals
**Description**: A comprehensive IDE for DuckDB that runs entirely in the terminal
**Repository**: https://github.com/tconbeer/harlequin
**Use Case**: SQL development, data analysis, database querying
**Key Pattern**: Full IDE experience in terminal (editor, query results, schema browser)

#### Datadog QA (ddqa)
**Purpose**: GitHub/Jira integration for QA workflows
**Description**: A tool for Jira users to perform QA of anticipated releases of code on GitHub
**Repository**: https://github.com/DataDog/ddqa
**Use Case**: Release management, quality assurance workflows
**Key Pattern**: Integration between multiple services (Jira/GitHub) in TUI

---

### AI & Chat Interfaces

#### Elia
**Purpose**: Terminal ChatGPT client
**Description**: A full-featured ChatGPT client built entirely with Textual
**Repository**: https://github.com/darrenburns/elia
**Use Case**: AI chat interface, conversational AI in terminal
**Key Pattern**: Chat interface with message history, streaming responses
**Notable**: Demonstrates complex conversational UI patterns in TUI

---

### Content Viewers & Browsers

#### Frogmouth
**Purpose**: Markdown viewer/browser
**Description**: A Markdown viewer and browser for your terminal (official Textualize project)
**Repository**: https://github.com/Textualize/frogmouth
**Use Case**: Documentation viewing, Markdown browsing, knowledge base navigation
**Key Pattern**: Document rendering, navigation, hyperlink handling
**Notable**: Official Textualize project showcasing rich content display

#### Baca
**Purpose**: E-book reader
**Description**: Open Source TUI e-book reader
**Repository**: https://github.com/wustho/baca
**Use Case**: Reading e-books in terminal
**Key Pattern**: Pagination, text rendering, reading mode interfaces

---

### System Monitoring & Analysis

#### Tiptop
**Purpose**: System monitoring tool
**Description**: Command-line system monitoring in the spirit of `top`. Displays various system stats with graphs. Cross-platform (all operating systems)
**Repository**: https://github.com/nschloe/tiptop
**Use Case**: System resource monitoring, performance analysis
**Key Pattern**: Real-time graphs, multi-metric displays, cross-platform compatibility

#### Flameshow
**Purpose**: Flamegraph viewer
**Description**: Terminal Flamegraph viewer for performance profiling
**Repository**: https://github.com/laixintao/flameshow
**Use Case**: Performance profiling, optimization analysis
**Key Pattern**: Complex graph visualization, interactive performance data

---

### DevOps & Infrastructure

#### Terraform textual UI (TFTUI)
**Purpose**: Terraform state viewer
**Description**: A powerful textual GUI that empowers users to effortlessly view and interact with their Terraform state
**Repository**: https://github.com/idoavrah/terraform-tui
**Use Case**: Infrastructure as Code management, Terraform state inspection
**Key Pattern**: Complex state visualization, infrastructure management UI

#### RecoverPy
**Purpose**: File recovery tool
**Description**: A powerful tool that leverages your system capabilities to recover lost files
**Repository**: https://github.com/PabloLec/RecoverPy
**Use Case**: File system recovery, data recovery workflows
**Key Pattern**: System interaction, file browsing, recovery operations

---

### Developer Experience Tools

#### Trogon
**Purpose**: Auto-generated TUI for CLI apps
**Description**: Auto-generate friendly terminal user interfaces for command line apps
**Repository**: https://github.com/Textualize/trogon
**Use Case**: CLI enhancement, command-line application UX improvement
**Key Pattern**: Automatic UI generation from CLI definitions
**Notable**: Official Textualize project - turns any CLI into interactive TUI

### Creative Applications

#### UPiano

## Common Patterns Across Showcase Apps

### Real-Time Data Display
- **Dolphie** (MySQL monitoring)
- **Tiptop** (system monitoring)
- **Harlequin** (query results)

**Pattern**: Live-updating widgets, refresh strategies, data streaming

### Complex Data Visualization
- **Flameshow** (flamegraphs)
- **Tiptop** (system graphs)
- **Harlequin** (query result tables)

**Pattern**: Custom visualization widgets, graph rendering, DataTable usage

### Content Rendering
- **Frogmouth** (Markdown)
- **Baca** (e-books)
- **Elia** (chat messages)

**Pattern**: Rich text rendering, pagination, scrollable content areas

### Multi-Panel Interfaces
- **Harlequin** (editor, results, schema)
- **Datadog QA** (issues, details, actions)
- **Terraform TUI** (resources, state, details)

**Pattern**: Split-screen layouts, dock containers, multiple coordinated views

### External Integration
- **Elia** (ChatGPT API)
- **Datadog QA** (Jira/GitHub)
- **Terraform TUI** (Terraform state)

**Pattern**: API integration, external service communication, async operations

## Key Insights for Developers

### Domain Versatility
Textual successfully serves diverse domains:
- Databases & data analysis
- System administration & DevOps
- Content consumption & media
- Creative applications
- AI/ML interfaces

### Complexity Range
From simple viewers (Baca) to complex IDEs (Harlequin), Textual scales to application complexity.

### Official vs Community
**Official Textualize Projects** (showcase framework capabilities):
- Frogmouth (Markdown browser)
- Trogon (CLI auto-generation)

**Community Projects** (demonstrate real-world adoption):
- 10 third-party applications across different domains

### Real-Time & Interactive Focus
Most showcase apps emphasize:
- Live data updates
- Interactive exploration
- Responsive user interfaces
- Complex state management
