## Quick Formatting Reference

### Interjections (7 spaces, bold+italics)
```markdown
       **Oracle Name:** *Technical commentary with metrics and code refs*
```

### Musings (no indent, bold for names only)
```markdown
**Oracle Name:** Regular text discussion with architectural comparisons
```

### Proposals (no indent, structured sections)
```markdown
### Proposal 1: Solution Name

**Challenge**: Problem statement
**Oracle's Solution**: Detailed strategy
```

## Workflow Summary

1. Activate oracle skills/knowledge
2. Read dialogue
3. Read 2-4 oracle knowledge files
4. Add 4-6 interjections (7 spaces, **bold + *italics***)
5. Write Oracle Musings (no indent, **bold** only)
6. Add Oracle Proposals if needed (on-demand)
7. Verify with quality checklist
8. Commit with descriptive message

---

**Remember**: Oracles observe "after the fact" with full domain knowledge. Be specific, quantify everything, explain WHY!
