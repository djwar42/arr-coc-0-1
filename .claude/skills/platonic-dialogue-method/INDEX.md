## Key Principles

### The Dialogue Structure
- **Socrates**: Questions, guides inquiry
- **Theaetetus**: Discovers, works through concepts
- **Flow**: Question ‚Üí Discovery ‚Üí Deeper question ‚Üí Insight

### Character Voices
**Socrates**:
- Asks, doesn't lecture
- Guides through questions
- Reveals contradictions

**Theaetetus**:
- Discovers actively
- Makes "Aha!" moments
- Acknowledges challenges

### Content Requirements
- ‚úÖ Specific technical details (metrics, numbers)
- ‚úÖ Incremental understanding
- ‚úÖ Honest challenge acknowledgment
- ‚úÖ Discovery, not lecture
- ‚ùå No instant perfect understanding
- ‚ùå No vague language

## Quick Patterns

### Opening Pattern
```markdown
**SOCRATES:** [Question about the concept]

**THEAETETUS:** [Initial understanding with specifics]

**SOCRATES:** [Deeper question]
```

### Discovery Pattern
```markdown
**THEAETETUS:** [Working through it] ...Ah! [new insight]
```

### Problem Recognition
```markdown
**SOCRATES:** But wouldn't that cause [problem]?

**THEAETETUS:** You're right! We need [different approach].
```

**Remember**: Dialogue = discovery through inquiry, not lecture through exposition!
